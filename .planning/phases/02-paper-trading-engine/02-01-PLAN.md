    2. Tao PaperMatchingEngine class (per D-01, D-02, D-03, D-04):
       - Class voi static methods (consistent voi RiskCalculator, CapitalService pattern)
       - calculateSlippage(price, quantity, avgDailyVolume, exchange):
         * liquidityTier: avgDailyVolume > 1M = HIGH (0.001), > 100K = MEDIUM (0.002), else LOW (0.003)
         * volumeImpact = quantity / max(avgDailyVolume, 1)
         * slippagePct = baseSpread * (1 + volumeImpact)
         * return Math.round(price * slippagePct)
       - calculateFillProbability(orderQty, avgDailyVolume):
         * ratio < 0.001 -> 0.95, < 0.01 -> 0.80, < 0.05 -> 0.50, < 0.10 -> 0.25, else 0.10
       - tryFillLimitOrder(order, currentPrice, avgDailyVolume):
         * Check price crosses: BUY if currentPrice <= limit_price, SELL if currentPrice >= limit_price
         * If not crossed: return { filled: false }
         * Roll fillProb: if Math.random() > fillProb return { filled: false, reason: 'LOW_FILL_PROBABILITY' }
         * LO fill tai limit price (khong slippage)
         * Return { filled: true, fillPrice: order.limit_price, slippage: 0 }
       - fillMarketOrder(order, currentPrice, avgDailyVolume, exchange):
         * Tinh slippage
         * BUY: fillPrice = currentPrice + slippage; SELL: fillPrice = currentPrice - slippage
         * Snap to tick size: snapToTickSize(fillPrice, exchange)
         * Clamp within price band (validatePriceInBand) -- per Pitfall 3
         * Return { filled: true, fillPrice, slippage }
       - fillATOOrder(order, openPrice): return { filled: true, fillPrice: openPrice, slippage: 0 }
       - fillATCOrder(order, closePrice): return { filled: true, fillPrice: closePrice, slippage: 0 }
       - Import snapToTickSize, validatePriceInBand tu tickSizeEngine.js
       - KHONG import slippageCalculator.js (do la cho SL/TP slippage, khac voi order fill slippage)
